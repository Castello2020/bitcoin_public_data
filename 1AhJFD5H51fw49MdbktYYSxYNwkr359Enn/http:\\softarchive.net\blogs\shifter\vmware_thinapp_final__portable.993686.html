VMWare ThinApp - a package designed for virtualization applications. Simply put, with the help of this tool you can from traditional applications to easily make a portable (portable) version. The approach allows us to "pack" the application in the container that contains all the information needed to work on any computer without having to install the desired application. At the same time, importantly, virtualized software in no way affect or alter the current state of the operating system. The very same idea is to prevent software conflicts that can disrupt the system and installed it on. Usefulness is obvious: developers, for example, may carry out tests of new products without risking the stability of the system environment. Approach will reduce the cost of deploying applications on end user computers. ThinApp allows you to run multiple versions of the same program. Thus, the simultaneous use of the Internet Explorer 6 and 7 should be useful in the process of testing sites.

In most cases ThinApp effectively implements its key task of virtualization through the mechanism of packaging applications and distribution systems, allowing application virtualization becoming available to end users on a network or using flash cards. This program is very good distributes applications, so no longer need to conduct extensive regression tests to check conflicts with other applications (or earlier versions of this product). Along with competing products (such as a beta version of Microsoft Application Virtualization 4.5 and Altiris Software Virtualization Solution) Program VMware ThinApp significantly different from the traditional tools for packaging applications because it can download an application package in a special section created in the system the end user, so that the application be done without crossing with other software that may already be installed.

Deliver and deploy applications were denied the opportunity to communicate with each other, since each of them performed in a separate "sandbox." If the user while working with virtualized Microsoft Word tries to open the Internet-link, a text editor can not turn to the browser. In ThinApp 4 has a new feature is Application Link, which is in the process of preparing the container, you can create the necessary links between virtualized applications. In the future, perhaps with the association of ThinApp Virtual Desktop Infrastructure, proposed use is located on a remote server system and application environment. Analysts suggest that by 2010 virtualization applications will be due the introduction of a large business - an approach will be adopted by 50% of the companies.

Features:

• Package your applications in standard EXE and MSI files
• Send your application with just one file without having to install
• Conversion of traditional applications in virtualized
• Opportunities to use the sandbox and the function of the image
• Detection of conflicts during the installation with application virtualization
• Elimination of threats and conflicts with other applications

Key features:

• Detection of conflicts during the installation with application virtualization
Application virtualization applications from the operating system and from each other, thereby reducing the cost of testing and conflicts from badly behaving applications. Just plug in an. MSI or. EXE for the implementation of a virtual system environment, including registry keys, DLL, third-party libraries and frameworks without requiring any installation of agents or applications on the underlying operating system.

• The only archiving and implementation at any point (Network, USB, Physical and virtual desktops)
Deploy virtualized applications in user mode, without administrative rights. No need to install device drivers and registry changes, as entire application and its virtual OS are delivered as a single EXE. Transparently stream large applications from a shared network drive without installation of client or server software. Upgrade or roll back applications while they work. The new version will be launched at next use.

• The use of mobile devices
Users can run applications on virtually any computer to which they have access, including computers in the kiosks and hotels. No need to install software or drivers, and do not need admin rights. Applications run directly from portable storage devices, including USB flash drives.

• Deliver software
Protects the local OS from modifications that might cause instability or breach security using your existing configuration management tools. Even restricted user accounts can safely run virtualized applications without requiring a local installation, making it more attractive to IT-staff, concerned about security.

• Easy migration of operating systems and applications
Application virtualization agentless ThinApp makes it easier to migrate operating systems, enabling applications to run on any operating system. Users can continue to operate as usual without disruption from problematic applications. Moreover, if the application needs to be removed from the endpoint - it will look like a simple file without the need for "clean" the registry.

What's New in This Release:

The ThinApp 4.7.2 release adds the following features to improve ThinApp usability, performance, and updates:

Streaming - ThinApp agent is now able to stream the virtualized application blocks to View desktops without deploying the entire package. For more information, see Knowledge Base article 2030247
Update Support - With this feature ThinApp can now enable Horizon Connector to recognize multiple versions of an application package and then deploy the updated package to the Agent. For more information, see Knowledge Base article 2030248
Relink Support - This feature allows ThinApp users to enable Horizon support for packages which are created prior to ThinApp 4.7 without a rebuild. For more information, see Knowledge Base article 2021928
New Sandbox Parameter - A new Package.ini parameter, RemoveSandBoxOnStart, is now available. You can set RemoveSandBoxOnStart = 1 to clear the sandbox on each launch of an application.
