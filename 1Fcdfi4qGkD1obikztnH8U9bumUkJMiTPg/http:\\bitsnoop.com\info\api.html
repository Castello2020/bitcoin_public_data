Bitsnoop public API interfaces

This is work in progress, page will be updated as more interfaces are made available. Stay tuned!

All interfaces are called via HTTP GET requests and will produce text/plain responses (unless stated othervise). UTF-8 is used where response can contain localized strings. UNIX time values are Central European (Summer) time. Responses are cached for 10 minutes. If you do more than 3 requests per second, your requests might be rate-limited.

We would be happy if you link to our site in return, there's an image available — .

Also if you want to link to specific torrent you can use shorthand URL http://bitsnoop.com/torrent/<infohash>, like this: http://bitsnoop.com/torrent/98C5C361D0BE5F2A07EA8FA5052E5AA48097E7F6.

A001 — FakeSkan status of torrent

Return validity (FakeSkan) status of torrent as known by our robots.

Entry point:

  • http://bitsnoop.com/api/fakeskan.php

Parameters:

  • hash — torrent info_hash value. Base32 encoding, case-insensitive;
  • json=1 — output result in JSON format;
  • img=1 — output image instead of text for use on your site (see example below);

Result

Torrent status:

ERROR hash not specified or internal error has occured. If you did specify hash, retry request in 10-15 minutes.
NOTFOUND torrent does not exist in index;
VERIFIED torrent is verified;
GOOD torrent has some "good" votes, not verified yet;
NONE no votes or too little of them to decide;
BAD some "bad" votes, not fake yet;
FAKE torrent is fake;

Examples:

Simple request:

http://bitsnoop.com/api/fakeskan.php?hash=98C5C361D0BE5F2A07EA8FA5052E5AA48097E7F6

Status images on your vebsite:

Insert code like this into your page's HTML:

<a href="http://bitsnoop.com/torrent/98C5C361D0BE5F2A07EA8FA5052E5AA48097E7F6" target="_new" title="Bitsnoop.com FakeSkan status">
<img border=0 src="http://bitsnoop.com/api/fakeskan.php?img=1&hash=98C5C361D0BE5F2A07EA8FA5052E5AA48097E7F6" width=148 height=18 /></a>

A002 — TrackerMatch records for torrent

Known trackers supporting specific torrent. Please note that the torrent itself may not be indexed by Bitsnoop fully.

Entry point:

  • http://bitsnoop.com/api/trackers.php
    • We recommend to use JSON format for this interface (i.e., "json=1" request parameter).

Parameters:

  • hash — torrent info_hash value. Base32 encoding, case-insensitive;
  • json=1 — output result in JSON format;

Errors

  • If we do not have scrape records for the torrent, we will return text "NOTFOUND".
  • If there's some sort of internal error, we will return text "ERROR" — in this case, retry request in several minutes.

Result

There are two modes of operation — plain-text and JSON.

Scrape records contain the following fields:

ANNOUNCE Announce URL for the tracker;
NUM_SEEDERS Number of seeds in last update;
NUM_LEECHERS Number of leechers in last update;
FOUND Unix timestamp — time when scrape record was first found;
UPDATED Unix timestamp — time when tracker was last accessed;

In JSON mode we return data like this:

Array( Array('ANNOUNCE' => 'http://...'), Array('ANNOUNCE' => 'http://...') ... )

In plain-text mode we will return text with header line and pipe-separated data for each tracker on a new line.
Lines are separated by Unix new line characted (ASCII 10). Like this:

#ANNOUNCE|NUM_SEEDERS|NUM_LEECHERS|FOUND|UPDATED
http://denis.stalker.h3q.com:6969/announce|181|95|1275896342|1275929285
http://tracker.openbittorrent.com/announce|182|74|1275888045|1275922616
http://tracker.publicbt.com/announce|227|93|1275888045|127592698

Request examples:

http://bitsnoop.com/api/trackers.php?hash=98C5C361D0BE5F2A07EA8FA5052E5AA48097E7F6

http://bitsnoop.com/api/trackers.php?hash=98C5C361D0BE5F2A07EA8FA5052E5AA48097E7F6&json=1

E001 — New Torrents, Last Hour

Lists new torrents indexed by Bitsnoop in the last hour. Output is cached for several minutes.

Format is simple — one torrent per line, pipe-separated infohash, name, categories, info URL and download URL. Unix line breaks.

There are two feeds available:

  • All torrents indexed in the last hour: http://bitsnoop.com/api/latest_tz.php?t=all
  • Only verified torrents indexed in the last hour: http://bitsnoop.com/api/latest_tz.php?t=verified

E002 — Full index data export, Daily

Each and every torrent in Bitsnoop index — free of charge! :-) Files are generated every day, at approximately 19:15 CET. Give or take.

Format is the same as in interface E001.

Two files are available:

  • All torrents (verified, fakes, unknown — everything) in Bitsnoop index: http://ext.bitsnoop.com/export/b3_all.txt.gz
  • Verified torrents in Bitsnoop index: http://ext.bitsnoop.com/export/b3_verified.txt.gz

E003 — Torrent trackers info export, Daily

Daily exports of all trackers our robots know about as well as of all torrents with seeding status and top 3 trackers for torrent.

Two files are exported, records are separated by Unix newline characters:

  • Tracker IDs and announce URLs, http://ext.bitsnoop.com/export/b3_e003_trackers.txt.gz, format is:
    • <ID number>|<Announce URL>
  • Torrent seeding status, http://ext.bitsnoop.com/export/b3_e003_torrents.txt.gz, format is:
    • <infohash>|<seeders>|<leechers>|<ID1>|<ID2>|<ID3>
    • ID1-ID3 are optional fields — respective IDs of top trackers for the torrent, as listed in trackers export file.
