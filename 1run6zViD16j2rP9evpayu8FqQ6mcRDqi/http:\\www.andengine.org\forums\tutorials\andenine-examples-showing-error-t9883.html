Re: AndEnine Examples Showing error

by alex » Mon Nov 19, 2012 12:48 am

I reproduced this just now. It seems the solution is:

At the top of HullAlgorithmExample.java, replace :

import org.andengine.entity.primitive.vbo.DrawMode;


with

import org.andengine.entity.primitive.DrawMode;



Next problem: TextBreakExample.java, line 106 should read:

this.mText = new Text(50, 40, this.mFont, "", 1000, new TextOptions(AutoWrap.LETTERS, AUTOWRAP_WIDTH, HorizontalAlign.CENTER, Text.LEADING_DEFAULT), vertexBufferObjectManager);



Next problem: BoundCameraExample.java, line 220 should read:

final AnimatedSprite face = new AnimatedSprite(pX, pY, this.mBoxFaceTextureRegion, this.getVertexBufferObjectManager());
face.animate(100);


Next problem: SplitScreenExample.java, line 179 should read:

final AnimatedSprite face = new AnimatedSprite(pX, pY, this.mBoxFaceTextureRegion, this.getVertexBufferObjectManager());
face.animate(100);



why do the examples have these basic bugs ?
