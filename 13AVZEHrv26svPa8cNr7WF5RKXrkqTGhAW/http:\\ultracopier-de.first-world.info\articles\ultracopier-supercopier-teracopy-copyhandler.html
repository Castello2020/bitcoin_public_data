Vergleich von ultracopier, supercopier, teracopy und copy handler


Ich wollte, dass mein Produkt abgeschlossen ist, vergleichen Sie alle Kopierer vorhandene Datei. Nehmen Sie es für eine persönliche Ansicht, ich also im Vergleich ultracopier, supercopier, teracopy und copyhandler

Hier wird kein:
  • besser als Teracopy
  • besser als SuperCopier
  • besser als CopyHandler
  • Es ist schnell
  • Es ist wunderschön
Ich vergleiche die Funktionalitäten und den tatsächlichen Wert, der nicht subjektiv.

Ultracopier


Die Vorteile der ultracopier in Bezug auf andere Kopierer meiner Meinung nach:
  • Multi-Plattform (linux, mac und windows)
  • Geschrieben in C + +, so dass jeder sich entwickeln
  • Erhältlich in vielen Versionen (normal, 64Bits, portable, debug)
  • Geschrieben in Qt für all die Vorteile dieser lib (Übersetzung, Themen, css, plugin, ...)
  • Versuchen Sie so viele Dinge wie ein standardisiertes und Pool bis der Code, und zu einem Maximum von lib intern und mit anderen Kopierern
  • Erhöhte Sicherheit für Datenübertragungen
  • Sehr gute Roadmap und die technischen Dokument oder vereinfachte, gute Gemeinschaft und sehr reaktionsschnell Entwickler (Ich habe Tools für hohe Leistung sofort eine Benachrichtigung)
  • Nice site maßgeschneiderte und eine Plattform, um eine große Gemeinschaft zu schaffen
  • Regelmäßig aktualisiert, aber SVN-Version erhältlich für die neueste Version noch nicht veröffentlicht
Nachteile:
  • Fehler in der Website (wir einfach nicht gut sein, überall), nicht sehr English English
  • Nur sehr wenige Spenden so kann ich nicht widme mich voll auf ultracopier
  • Einige Übersetzungen werden über Google Translator getan, aber die Übersetzung System haben, das zu ändern erleichtert

Supercopier


Die Vorteile der SuperCopier vis-à-vis anderen Kopierern meiner Meinung nach:
  • Langjährige
  • Gehen Sie unter Wein, wenn Sie die Kopien mit der Hand werfen
  • Open source
Nachteile:
  • Geschrieben in Pascal, können nur wenige Entwickler helfen

Teracopy


Die Vorteile gegenüber anderen Kopierern TeraCopy meiner Meinung nach:
  • Anpassung Kopiereinstellungen in Abhängigkeit von der Medien-Quelle und Ziel
Nachteile:
  • Gebührenpflichtig
  • Kleine Änderung
  • Keine Optionen für die Kopie Fenster
  • Funktioniert nicht unter Windows 8 zu arbeiten (Zuverlässigkeit und schlechter Prävention von Bug / Crash)
  • Dass unter windows
  • Utility, um die Größe des Puffers durch den Schieber an der Schnittstelle gezeigt, zu verwalten?

Copy handler


Die Vorteile einer Kopie Handler in Bezug auf andere Kopierer meiner Meinung nach:
  • Geschrieben in C + +, so dass jeder entwickeln kann, aber einige spezielle Windows und Visual Studio obligatorisch
  • ist effizient
  • Kleine Fenster in der Nähe der Systray, die besser geeignet sein für manche
Nachteile:
  • Nur unter Windows und hatte Probleme mit neuen Fenstern, aber das wird korrigiert
  • Alt genug, Version (meist wurde Mitte 2009), nur gepflegt, keine neue Version geplant
  • Schnittstelle genug Ofen und alle notwendigen Informationen für Entwickler, die optimieren wollen
  • Es gibt keine portable Version
